# OmniRAG: Точный локальный AI-ассистент

Проект представляет собой оптимизированную локальную RAG-систему (Retrieval-Augmented Generation), которая позволяет вести диалог по содержимому пользовательских документов (PDF, DOCX, TXT, HTML, XML). Система разработана с прицелом на строгую фактологию и работает максимально стабильно даже на небольших языковых моделях (например, Qwen 2.5 3B).

### Основные архитектурные особенности
- **Режим "Строгого редактора"**: LLM-модель работает с температурой 0.0 (нулевая креативность), что исключает галлюцинации и выдумывание фактов. Ответы генерируются строго по найденному тексту.
- **Продвинутый поиск (Hybrid Search + Reranker)**: В качестве векторной базы используется **Qdrant**. Система делает гибридный поиск (по смыслу и по ключевым словам BM25), извлекая 15 релевантных кусков. Затем сверхлегкий **Cross-Encoder** (работающий даже на CPU) перепроверяет их и оставляет только 4 самых точных куска. Это защищает модель от проблемы "Lost in the Middle" и не допускает потерю данных.
- **Обогащение метаданными (Metadata Enrichment)**: Имя исходного документа "вшивается" в каждый его текстовый фрагмент, что в разы повышает точность поиска по узким запросам (например, когда вопрос касается конкретной книги или автора).
- **Умное окно памяти**: Система запоминает только последние 2 сообщения диалога (1 вопрос и 1 ответ). Это дает возможность задавать уточняющие вопросы ("А что насчет этого?"), при этом защищая модель от переполнения контекста старым текстом.
- **Временные файлы**: Возможность загружать документы через пользовательский веб-интерфейс только на время текущей сессии. Как только страница перезагружается, эти данные удаляются из оперативной памяти.
- **Полностью локальная обработка**: Вся обработка (эмбеддинги, векторизация, генерация Ollama) происходит на устройстве пользователя без передачи данных в интернет. Код автоматически задействует GPU (видеокарту) при наличии, а при отсутствии плавно переключается на CPU.

## Краткое руководство по запуску

Система поддерживает два варианта запуска: полностью изолированный (всё внутри Docker) и гибридный (нейросеть работает напрямую в Windows для обхода ограничений WSL2 и достижения максимальной скорости).

### Вариант А: Полностью в Docker (Классический)
Подходит для серверов или машин с идеальной настройкой Docker GPU.

1. Поместите файлы для базы знаний в папку `data/`.
2. Запустите контейнеры:
   - Для GPU: `docker-compose --profile gpu up -d`
   - Для CPU: `docker-compose --profile cpu up -d`
3. Скачайте модель (только при первом запуске):
   `docker exec ollama_db_gpu ollama pull qwen2.5:3b` *(замените `gpu` на `cpu` если нужно)*
4. Проиндексируйте файлы:
   `docker exec -it pharma_rag_ui_gpu python -m src.cli.ingest`
5. Откройте интерфейс: `http://localhost:8501`

### Вариант Б: Гибридный (Native Windows Ollama) - РЕКОМЕНДУЕТСЯ ДЛЯ WINDOWS
Этот вариант убирает бутылочное горлышко WSL2-виртуализации и позволяет видеокарте работать на 100% мощности.

1. Установите **Ollama** для Windows с сайта [ollama.com/download](https://ollama.com/download).
2. Запустите консоль (CMD) в Windows и скачайте модель: `ollama run qwen2.5:3b`.
3. Запустите базу данных и интерфейс через специальный файл:
   `docker-compose -f docker-compose.windows.yml --profile gpu up -d`
4. Проиндексируйте вашу базу знаний (папку `data/`):
   `docker exec -it pharma_rag_ui_gpu python -m src.cli.ingest`
5. Откройте интерфейс: `http://localhost:8501`

*(В этом режиме интерфейс из Docker подключается к быстрой локальной Ollama по адресу `host.docker.internal`)*.

### Остановка проекта
Для завершения работы всех контейнеров выполните команду:
```bash
# Если запускали Вариант А
docker-compose --profile gpu down --remove-orphans

# Если запускали Вариант Б
docker-compose -f docker-compose.windows.yml --profile gpu down --remove-orphans
```
